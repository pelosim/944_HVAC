## GPIO Pin Assignments

All pin assignments are at the top of `hvac_backend.py` (around lines 40–55).

```python
# ── Relay Outputs (active-LOW: GPIO LOW = relay ON) ──
PIN_BLOWER_HI   = 5      # Fan high speed
PIN_BLOWER_LOW  = 6      # Fan low speed
PIN_AC_CLUTCH   = 18     # A/C clutch command to MS3 (RLY1)
PIN_HEAT_VALVE  = 19     # Heating valve solenoid (RLY2)
PIN_OUTSIDE_AIR = 26     # Outside air solenoid (RLY3)

# ── H-Bridge Outputs (active-HIGH: GPIO HIGH = motor runs) ──
PIN_MIX_COLD    = 23     # Mixing flap → COLD direction
PIN_MIX_HOT     = 24     # Mixing flap → HOT direction
PIN_DEF_IN1     = 16     # Defrost flap direction 1
PIN_DEF_IN2     = 20     # Defrost flap direction 2
PIN_FOOT_IN1    = 12     # Footwell flap direction 1
PIN_FOOT_IN2    = 21     # Footwell flap direction 2
```

### To change a pin:
1. Open `hvac_backend.py` in nano or VS Code
2. Change the number (e.g., `PIN_BLOWER_HI = 5` → `PIN_BLOWER_HI = 13`)
3. Restart: `python3 hvac_backend.py`

### To add a new output:
1. Add a new constant: `PIN_MY_NEW_RELAY = 25`
2. Add to `_init_gpio()` → add the pin to the relay or H-bridge setup list
3. Add a control method like `set_my_relay(self, on: bool)`
4. Call it from the `tick()` method based on your logic
5. Add a state field to `HVACState` so the dashboard can see it

```python
ADS_I2C_ADDR       = 0x48

ADS_DEF_CHANNEL    = 1    # CH1 → defrost flap feedback
ADS_FOOT_CHANNEL   = 2    # CH2 → footwell flap feedback

```

### Analog scaling (line ~68):
```python
ADC_MV_MIN = 225     # mV at 0% position (flap fully one way)
ADC_MV_MAX = 4090    # mV at 100% position (flap fully other way)
```

### To swap or change a channel:
Just change the channel number (0–3). Restart backend.

### To use the spare CH3:
1. Add: `ADS_NEW_CHANNEL = 3`
2. The `_init_ads()` method already reads all enabled channels
3. Add `read_flap_position(ADS_NEW_CHANNEL)` in `tick()`

## PID Loop Tuning

All PID gains are at the top of `hvac_backend.py` (around lines 72–83):

```python
# ── Flap Position PIDs (controls how H-bridges drive motors) ──
FLAP_PID_KP = 2.0      # Proportional: how hard to drive toward target
FLAP_PID_KI = 0.5      # Integral: eliminates steady-state error
FLAP_PID_KD = 0.1      # Derivative: dampens overshoot
FLAP_DEADBAND = 2.0    # %: motor stops within this band of target

# ── Temperature PID (drives mixing flap setpoint from temp error) ──
TEMP_PID_KP = 3.0      # How aggressively to adjust flap for temp error
TEMP_PID_KI = 0.3      # Eliminates offset over time
TEMP_PID_KD = 0.5      # Reacts to rapid temp changes
TEMP_DEADBAND = 1.0    # °F: don't adjust within this tolerance
```

### Tuning procedure:

**Flap PIDs (start here — tune on the car):**
1. Set Ki=0, Kd=0, start with Kp=2.0
2. Command a flap to 50% — does it get there?
3. If too slow → increase Kp
4. If it oscillates back and forth → reduce Kp by 30%
5. Add Ki=0.1, increase slowly until steady-state error is gone
6. Add Kd=0.05 if there's overshoot
7. Increase deadband if the motor hunts (chatters near target)

**Temperature PID (tune after flaps work):**
1. Set a setpoint of 75°F with heat on
2. Watch duct temp — does it approach 75°F?
3. If too slow → increase TEMP_PID_KP
4. If it overshoots wildly → reduce Kp, increase Kd

### Live tuning via REST API:
```bash
# Check current state
curl http://192.168.1.49:8000/api/state | python3 -m json.tool

# Send commands
curl -X POST http://192.168.1.49:8000/api/command \
  -H "Content-Type: application/json" \
  -d '{"setpoint_f": 75, "fan_speed": "HI"}'
```

## Simulation vs Hardware Mode

The backend **auto-detects** at startup:

| You see this in terminal | Meaning |
|-------------------------|---------|
| `HVAC controller started — SIMULATION mode` | RPi.GPIO not found; fake sensor values, no real outputs |
| `HVAC controller started — HARDWARE mode` | Real GPIO, real sensors |
| `GPIO initialized` | Real GPIO is working |
| `ADS1115 init failed` | ADC not detected (flap feedback won't work, but everything else does) |
| `Found DS18B20: xxxxxxxx` | Temperature sensor detected |
| `No DS18B20 sensors found` | No temp sensors on 1-Wire bus |

### To force simulation mode on the Pi:
Add this near the top of `hvac_backend.py`, right after the `try/except ImportError` block:

```python
SIMULATE = True   # Force simulation even if GPIO is available
```

### To test on your Mac (always simulates):
```bash
pip install fastapi "uvicorn[standard]"
cd ~/hvac   # or wherever you have the files
python3 hvac_backend.py
# Open http://localhost:8000
```

## Dashboard Layout Changes

The dashboard is in `HVACDashboard.jsx`. The layout uses a 4-column CSS grid:

```
┌──────────┬────────────┬────────────┬──────────┐
│  ZONE 1  │   ZONE 2   │   ZONE 3   │  ZONE 4  │
│  300px   │    1fr     │    1fr     │  300px   │
│          │            │            │          │
│ Gauges   │ Setpoint   │ Schematic  │ Controls │
│ Status   │ Mode pill  │ Flap bars  │ Toggles  │
│ Clock    │ Vent modes │            │ MAX A/C  │
│          │            │            │ Fan spd  │
└──────────┴────────────┴────────────┴──────────┘
             1920px total (for 1920×720 display)
```

### Grid columns (line ~406):
```javascript
gridTemplateColumns: "300px 1fr 1fr 300px"
```
Change `300px` to make side panels wider/narrower.

### To add a new gauge:
In ZONE 1 (or wherever), add:
```jsx
<ArcGauge value={myValue} min={0} max={100} size={140}
  label="MY LABEL" unit="°F" color={C.cyan} thickness={5} />
```

### To add a new toggle button:
In ZONE 4, add inside the toggles `<div>`:
```jsx
<Toggle label="MY BTN" icon="⚡" active={myState}
  onToggle={() => setMyState(!myState)} color={C.cyan} />
```
And add the state variable at the top:
```javascript
const [myState, setMyState] = useState(false);
```

### To add a new status indicator:
In the System Status panel (ZONE 1), add:
```jsx
<StatusDot label="MY STATUS" active={someCondition} color={C.green} />
```

### Color reference:
```javascript
C.cyan   = "#00fdff"    // Primary accent
C.warm   = "#ff6633"    // Heat/hot indicators
C.cool   = "#2288ff"    // A/C/cold indicators
C.green  = "#00cc66"    // OK/active status
C.danger = "#ff2244"    // Error/off status
C.text   = "#e0e4ef"    // Bright text
C.textMid = "#8892b0"   // Labels (readable)
C.textDim = "#556080"   // Subtle text
```

## Service Management

Once you're done testing and want auto-start at boot:

```bash
# Create the service (one time)
sudo systemctl enable hvac
sudo systemctl start hvac

# Day-to-day
sudo systemctl restart hvac      # After code changes
sudo systemctl stop hvac         # Stop backend
sudo systemctl status hvac       # Check if running
journalctl -u hvac -f            # Live logs
journalctl -u hvac --since today # Today's logs
```

## Hardware Diagnostic Commands

```bash
# I²C — should show 48 for ADS1115
i2cdetect -y 1

# 1-Wire — list temp sensors
ls /sys/bus/w1/devices/28-*

# Read a raw temperature
cat /sys/bus/w1/devices/28-000000bd3d51/temperature
# Divide by 1000 for °C, then convert to °F

# Test a single GPIO output (careful — this drives real hardware!)
python3 -c "
import RPi.GPIO as GPIO
GPIO.setmode(GPIO.BCM)
GPIO.setup(24, GPIO.OUT)
GPIO.output(24, GPIO.HIGH)   # Drive mixing flap HOT
import time; time.sleep(0.5)
GPIO.output(24, GPIO.LOW)    # Stop
GPIO.cleanup()
"

# Check what's using port 8000
sudo lsof -i :8000

# Check if Node-RED is still running (should be disabled)
systemctl status nodered
```
